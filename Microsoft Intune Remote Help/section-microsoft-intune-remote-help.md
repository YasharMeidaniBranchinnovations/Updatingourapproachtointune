# Microsoft Intune Remote Help

## Objective

Describe the use of Microsoft Intune Remote Help to enable secure, cloud-based remote assistance for end users while maintaining centralized control, licensing compliance, and role-based access.

## High Level End to End Workflow

Remote Help allows authorized IT personnel to remotely view or control a user’s Windows device after an explicit user request and approval. The feature relies on Intune licensing, role-based access control, and Microsoft cloud services. Once enabled, Remote Help sessions are initiated using a secure, time-limited connection code and are brokered through Microsoft services over HTTPS.

## Feature Overview

Remote Help is an Intune add-on capability that enables help desk operators or IT administrators to provide live assistance to users. A user requests help, an authorized operator connects using a generated code, and the user approves the session before screen sharing or control begins. All actions are performed securely without exposing inbound network ports.

## Platform and Prerequisites

Remote Help currently supports Windows 10 and Windows 11 devices. Devices must be enrolled in Intune. Users and operators must have an active Intune subscription and an additional Remote Help add-on license or an Intune Suite license. Network connectivity over HTTPS on port 443 must be available, and Microsoft Remote Help service endpoints must not be blocked by firewalls or network security controls.

## Licensing Model

Remote Help is not included with Intune Plan 1 or Plan 2 by default. It is available either as a standalone Remote Help add-on license or as part of the Intune Suite. Licenses must be assigned to both the user requesting help and the operator providing assistance. Licensing status is managed through the Microsoft 365 admin center.

## Remote Help Enablement

Remote Help must be explicitly enabled at the tenant level in the Intune admin center. This configuration is performed under Tenant Administration in the Remote Help settings. Administrators can enable or disable Remote Help and configure options such as allowing help on unenrolled devices and enabling or disabling chat functionality.

## Role and Access Configuration

Access to Remote Help is controlled using Intune role-based access control. Operators must be assigned an appropriate role, typically the Help Desk Operator role, which includes permissions for remote assistance actions such as viewing the screen or taking full control. Role assignments are applied through admin groups, and scope groups define which users or devices operators are allowed to assist. Exclusions always override inclusions when both are present.

## Operator Group Configuration

A dedicated security group is created in Microsoft Entra ID for Remote Help operators. This group is assigned the Help Desk Operator role through Intune role assignments. Membership in this group determines which users are permitted to initiate Remote Help sessions as operators.

## Client Application Deployment

Remote Help requires the Remote Help client application to be installed on both the operator and user devices. The application is downloaded from Microsoft’s official documentation and installed locally using a standard installation wizard. The application requires user authentication before use.

## Session Initiation and User Consent

Remote Help sessions are initiated using a one-time code generated by the operator. The user enters this code into the Remote Help application and explicitly approves the session. Operators may request view-only access or full control, and users must consent before control is granted. Sessions cannot begin without user approval.

## Session Behavior and Connectivity

Remote Help sessions are brokered through Microsoft cloud services using HTTPS over port 443. No inbound firewall ports or remote desktop services are required. This eliminates the need for traditional port forwarding or network address translation configurations used by legacy remote support tools.

## Device Level Integration

Remote Help can be launched directly from the Intune admin center by selecting a managed device and initiating a new remote assistance session. This provides a centralized method for operators to assist users without manually opening the application, though direct application launch is also supported.

## Operational Considerations

Remote Help operates independently of traditional domain join requirements. Devices do not need to be hybrid or Azure AD joined to use Remote Help, provided they are enrolled and licensed appropriately. Session performance may vary based on device resources and network conditions.

## Summary

Microsoft Intune Remote Help provides a secure, cloud-based method for delivering remote assistance to Windows devices. The feature requires additional licensing, explicit tenant enablement, and role-based access control. Sessions are user-approved, encrypted, and delivered over standard HTTPS without exposing network ports. Remote Help simplifies support operations while maintaining strong security and administrative control.
