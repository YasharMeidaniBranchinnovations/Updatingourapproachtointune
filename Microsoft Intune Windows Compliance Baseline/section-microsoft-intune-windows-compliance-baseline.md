## Microsoft Intune Windows Compliance Baseline

### Objective

Establish a standardized Windows compliance baseline in Microsoft Intune that evaluates device security posture and readiness before access to organizational resources is permitted.

### Overview

Microsoft Intune compliance policies define evaluation rules that determine whether a managed device meets required security and trust standards. These policies do not configure devices. Instead, they assess the current state of the device and classify it as compliant or noncompliant based on defined criteria.

For Windows devices, compliance is implemented as a single logical baseline composed of multiple targeted compliance policies. Each policy evaluates a specific security domain. All evaluation domains must pass for a device to be considered compliant.

The baseline is supported by **JSON based compliance policy definitions that are maintained as ready to import artifacts**. These files are stored alongside the documentation and can be directly imported into Microsoft Intune to deploy the baseline consistently across environments.

### Platform Scope

This compliance baseline applies to devices running Windows 10 and later. Other operating systems require separate compliance baselines tailored to platform specific capabilities and risks.

### High Level End to End Workflow

Windows compliance policies are created in the Intune admin center or imported using the provided JSON files and assigned to targeted user or device groups. Managed Windows devices periodically synchronize with Intune and are evaluated against the defined compliance criteria. Compliance status is reported to Intune and consumed by Conditional Access policies to enforce access decisions.

### Evaluation Domains

The Windows compliance baseline evaluates device trust and security across the following domains:

**User authentication**

Local access protection is validated by enforcing password and lock requirements. Devices must use a compliant password configuration and automatically lock after a defined period of inactivity.

**Device health and boot integrity**

Platform trust is established by validating BitLocker encryption, Secure Boot, and code integrity. These checks ensure the device boots into a trusted and tamper resistant state.

**Endpoint security posture**

Core security controls are validated, including firewall status, antivirus and antispyware protection, and the presence of trusted hardware such as TPM. These checks ensure baseline protection against common threats.

**Malware protection health**

Microsoft Defender is evaluated to confirm that it is enabled, actively protecting the device, and using up to date security intelligence. This ensures continuous protection against malware and emerging threats.

### Minimum Operating System Version

A minimum Windows operating system version may be enforced to ensure devices remain on supported builds that receive security updates and maintain compatibility with modern protection features.

### Noncompliance Handling

If a device fails one or more compliance checks, it is marked as noncompliant. Intune records the noncompliant state and can notify the user. Enforcement timing and grace periods may vary by evaluation domain based on risk sensitivity.

Compliance policies alone do not block access to resources.

### Integration with Conditional Access

Conditional Access policies consume the compliance signal generated by Intune. When compliance is required by Conditional Access, noncompliant devices are blocked from accessing organizational resources such as Microsoft 365, SharePoint, and OneDrive until remediation occurs.

### Assignment and Targeting

Compliance policies may be assigned to users, devices, or both. User based assignment allows compliance to follow the user across multiple devices, while device based assignment targets specific machines such as kiosks or shared systems. Exclusions may be configured, and exclusion assignments take precedence.

### Evaluation and Reporting

Devices evaluate compliance during regular Intune synchronization cycles. Administrators may trigger manual syncs to accelerate evaluation. Compliance results are visible in the Intune admin center and available for reporting, auditing, and access control decisions.

### Summary

This Windows compliance baseline provides a structured and scalable approach to evaluating device security and trust in Microsoft Intune. The baseline is fully documented and backed by **JSON ready compliance policy files available within the accompanying folder**, enabling repeatable deployment, version control, and consistent enforcement across environments.
